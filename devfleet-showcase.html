// The Problem

AI 很聪明，
但只有一双手

AI 编码工具已经能独立完成复杂任务，但它们都被困在单台机器上。你有3台电脑，AI 只能用1台。

🏝️

设备孤岛

每台设备上的 AI 编码工具互不通信。办公电脑、家用台式、云服务器——各自为战，无法协同。

⏳

串行等待

前端改完才能改后端，后端改完才能写测试。明明有3台电脑可以同时干活，却只能一台一台排队。

🤷

手动合并

即使你手动在每台电脑上分别启动 AI，最终还是要自己 Git merge。冲突处理、分支管理全靠人肉。

// How It Works

AI 指挥，
舰队执行

AI 通过 MCP 协议调用 DevFleet 工具，自己决定怎么拆任务、分给谁、何时合并。

1

AI 分析任务，自主拆分

AI 理解需求后，自己决定如何将任务拆分为可并行的子任务。不是按换行符傻拆，而是真正理解代码结构。

// AI 自主决策，不是规则拆分
dispatch_task({
  description: "重构认证模块",
  device: "office-pc"
})
2

多设备并行执行

任务通过 WebSocket 实时分发到在线设备。每台设备在自己的 Git 分支上独立工作，互不干扰。

office-pc → devfleet/trae/sub-1-a3f2
home-mac  → devfleet/cursor/sub-2-b7e1
cloud     → devfleet/claude/sub-3-c9d4
3

自动合并分支

所有设备完成后，AI 调用合并工具自动 fetch、merge、push。冲突时回滚并报告，不会破坏代码库。

merge_task({ task_id: "task-001" })
// ✓ fetch → merge → push，全自动

// MCP Integration

7 个 MCP 工具，
AI 的舰队操控台

标准 MCP 协议，兼容 Trae、Cursor、Claude Code、Codex。AI 像调用函数一样调度你的设备舰队。

devfleet_list_devices

查看舰队状态

获取所有在线设备及其工具状态，AI 据此决定任务分配策略。

devfleet_dispatch_task

派发任务

向指定在线设备派发 Git 代码任务，设备自动克隆仓库、创建分支、开始执行。

devfleet_get_task

查询任务进度

读取任务状态、日志、分支信息和完成进度，实时掌握舰队动态。

devfleet_wait_for_task

等待任务完成

阻塞等待多设备任务全部完成，支持超时保护。AI 可同时等待多个并行任务。

devfleet_merge_task

合并分支

在主设备上真实执行 fetch、merge、push。冲突时自动 abort 并报告，安全可靠。

devfleet_next_task

获取待执行任务

工作设备上的 AI Agent 读取当前分配给自己的任务，开始执行。

devfleet_report_task_progress

回写执行进度

工作设备上的 AI Agent 回报阶段性进度和日志，完成后自动提交推送。
