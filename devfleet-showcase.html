// The Problem

AI 很聪明，
但只有一双手

AI 编码工具已经能独立完成复杂任务，但它们都被困在单台机器上。你有3台电脑，AI 只能用1台。

🏝️

设备孤岛

每台设备上的 AI 编码工具互不通信。办公电脑、家用台式、云服务器——各自为战，无法协同。

⏳

串行等待

前端改完才能改后端，后端改完才能写测试。明明有3台电脑可以同时干活，却只能一台一台排队。

🤷

手动合并

即使你手动在每台电脑上分别启动 AI，最终还是要自己 Git merge。冲突处理、分支管理全靠人肉。

// How It Works

AI 指挥，
舰队执行

AI 通过 MCP 协议调用 DevFleet 工具，自己决定怎么拆任务、分给谁、何时合并。

1

AI 分析任务，自主拆分

AI 理解需求后，自己决定如何将任务拆分为可并行的子任务。不是按换行符傻拆，而是真正理解代码结构。

// AI 自主决策，不是规则拆分
dispatch_task({
  description: "重构认证模块",
  device: "office-pc"
})
2

多设备并行执行

任务通过 WebSocket 实时分发到在线设备。每台设备在自己的 Git 分支上独立工作，互不干扰。

office-pc → devfleet/trae/sub-1-a3f2
home-mac  → devfleet/cursor/sub-2-b7e1
cloud     → devfleet/claude/sub-3-c9d4
3

自动合并分支

所有设备完成后，AI 调用合并工具自动 fetch、merge、push。冲突时回滚并报告，不会破坏代码库。

merge_task({ task_id: "task-001" })
// ✓ fetch → merge → push，全自动

// MCP Integration

9 个 MCP 工具，
AI 的舰队操控台

标准 MCP 协议，兼容 Trae、Cursor、Claude Code、Codex。含 Computer Use 自动向 Trae 输入框写入任务。

devfleet_list_devices

查看舰队状态

获取所有在线设备及其工具状态，AI 据此决定任务分配策略。

devfleet_dispatch_task

派发任务

向指定在线设备派发 Git 代码任务，设备自动克隆仓库、创建分支、开始执行。

devfleet_get_task

查询任务进度

读取任务状态、日志、分支信息和完成进度，实时掌握舰队动态。

devfleet_wait_for_task

等待任务完成

阻塞等待多设备任务全部完成，支持超时保护。AI 可同时等待多个并行任务。

devfleet_merge_task

合并分支

在主设备上真实执行 fetch、merge、push。冲突时自动 abort 并报告，安全可靠。

devfleet_next_task

获取待执行任务

工作设备上的 AI Agent 读取当前分配给自己的任务，开始执行。

devfleet_computer_use_start_trae_task

Computer Use 发任务

自动打开 Trae 工作区、点击「新任务」并写入 prompt，无需手动复制粘贴。

# DevFleet E2E 最小闭环 — AI 必须按序调用，用户只说一句话
# 用户: 「用 DevFleet 跑本机 E2E：README 追加测试行，Trae 改码，merge 到 main」

1. devfleet_list_devices()
   → 确认 Home Mac online + devTool=trae
   → 离线则报错并提示启动 DevFleet Agent

2. devfleet_dispatch_task({
     title: "E2E: README 追加测试行",
     repo_url: "http://127.0.0.1:8765/bare.git",
     base_branch: "main",
     device_id: "<online-trae-device>",
     prompt: "在 README.md 末尾追加一行：\n\n<!-- devfleet-e2e test line -->"
   })

3. devfleet_computer_use_start_trae_task({
     prompt: "<同上 prompt>"
   })
   # 自动: activate Trae CN → 点「新任务」→ 粘贴 → Enter
   # 禁止让用户手动复制 prompt

4. # Trae Agent 在 agent-workspace 改码、commit、push 子分支
   # AI 不介入，只等待

5. devfleet_wait_for_task({ task_id, timeout_ms: 600000 })
   → status=completed 后继续

6. devfleet_merge_task({
     task_id,
     workspace_path: "/tmp/devfleet-e2e/merge-workspace",
     push: true
   })
   → 汇报各步耗时 + merge commit hash
